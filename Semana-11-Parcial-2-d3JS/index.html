introducción

1. ¿Qué es D3.js?

D3.js (Data-Driven Documents) no es una librería de gráficos: es un conjunto de herramientas para unir datos a elementos del DOM. Tú decides qué elementos crear y cómo deben verse; D3 te da las herramientas para calcularlo a partir de los datos.

D3 vs Chart.js: Chart.js dibuja gráficos predefinidos (barra, línea, pie…) con pocas líneas de código. D3 no dibuja nada directamente: te da escalas, ejes y generadores de formas para que construyas cualquier gráfico imaginable. Más código, control infinito.

La filosofía central: Join

Todo en D3 gira en torno al concepto de data join: vincular un array de datos a un conjunto de elementos DOM. D3 calcula automáticamente qué elementos hay que crear (enter), actualizar (update) o eliminar (exit). 

// La idea central de D3 en 6 líneas:

d3.select('svg')              // selecciona el SVG
  .selectAll('rect')        // selecciona todos los  (aunque no existan aún)
  .data([10, 25, 40, 15, 30])  // vincula el array de datos
  .join('rect')              // crea un  por cada dato (enter + update)
    .attr('width', 40)       // atributo fijo
    .attr('height', d => d * 4) // atributo que depende del dato (d)
    .attr('fill', 'steelblue');  // color
selecciones

2. Selecciones y Join de datos

Las selecciones son el corazón de D3. Funcionan como querySelectorAll pero devuelven un objeto D3 con métodos para vincular datos y modificar el DOM. 

// ── SELECCIONAR ELEMENTOS ────────────────────────────────────
d3.select('#mi-svg')         // selecciona UNO (el primero que coincida)
d3.selectAll('.barras rect') // selecciona TODOS

// ── JOIN DE DATOS (patrón moderno con .join()) ───────────────
const datos = [42, 18, 73, 55, 29];

svg.selectAll('circle')
  .data(datos)               // vincula datos
  .join('circle')            // crea/actualiza/elimina elementos
    .attr('r', d => d / 2)  // radio proporcional al valor (d)
    .attr('cx', (d, i) => i * 80 + 40) // i = índice del dato
    .attr('cy', 50)
    .attr('fill', '#f68e2e');

// ── MODIFICAR ATRIBUTOS vs ESTILOS ───────────────────────────
sel.attr('fill', 'red');     // atributo SVG: fill, cx, r, d, ...
sel.style('color', 'red');  // propiedad CSS inline
sel.text('Hola');           // contenido de texto
sel.html('<b>Ok</b>');      // contenido HTML
sel.classed('activo', true); // añadir/quitar clase CSS
sel.property('value', 'x'); // propiedad DOM (value, checked...)

escalas

3. Escalas

Las escalas son funciones que mapean un dominio de datos (los valores reales) a un rango visual (píxeles, colores...). Son la pieza más importante de D3: sin escalas, no hay forma de convertir datos en posiciones.

EscalaDominio → RangoUso típico
d3.scaleLinear()Número → Número/ColorEje Y, radios, colores continuos
d3.scaleBand()Categoría → NúmeroEje X de gráficos de barras
d3.scaleOrdinal()Categoría → ColorColores por categoría
d3.scaleTime()Fecha → NúmeroSeries temporales
d3.scaleLog()Número (log) → NúmeroDatos con rangos muy amplios
d3.scaleSqrt()Número (√) → NúmeroRadios de burbujas (área ∝ dato)
 
// ── scaleLinear: dominio continuo → rango continuo ───────────
const escalaY = d3.scaleLinear()
  .domain([0, 100])    // mínimo y máximo de los datos
  .range([altura, 0]); // píxeles (invertido: SVG crece hacia abajo)

escalaY(50);  // → altura / 2 (en píxeles)
escalaY(0);   // → altura (fondo del SVG)
escalaY(100); // → 0 (tope del SVG)

// ── scaleBand: categorías en posiciones discretas ────────────
const meses = ['Ene', 'Feb', 'Mar', 'Abr', 'May'];
const escalaX = d3.scaleBand()
  .domain(meses)        // categorías
  .range([0, ancho])    // espacio total en píxeles
  .padding(0.2);       // espacio entre barras (0-1)

escalaX('Feb');           // → posición X de la barra Febrero
escalaX.bandwidth();      // → ancho de cada barra

// ── scaleOrdinal: categorías → colores ───────────────────────
const color = d3.scaleOrdinal(d3.schemeTableau10)
  .domain(categorias);
color('Ventas');  // → '#4e79a7' (color de esa categoría)

// ── Métodos útiles ────────────────────────────────────────────
d3.extent(datos);       // → [min, max] del array
d3.max(datos);          // → valor máximo
d3.min(datos, d => d.valor); // → mínimo usando accessor

gráfico de barras

4. Gráfico de barras — el "hola mundo" de D3

El gráfico de barras es el punto de partida para aprender D3. Introduce todos los conceptos fundamentales: SVG con márgenes, dos escalas, dos ejes y los rectángulos vinculados a datos. 

// ── PATRÓN MARGIN CONVENTION (convención estándar de D3) ─────
// Siempre se trabaja con un SVG exterior y un  interior.
const margen = { top: 20, right: 20, bottom: 40, left: 50 };
const ancho   = 500 - margen.left - margen.right; // ancho del área de dibujo
const altura  = 300 - margen.top  - margen.bottom;

const svg = d3.select('#mi-grafico')
  .append('svg')
  .attr('width',  ancho  + margen.left + margen.right)
  .attr('height', altura + margen.top  + margen.bottom)
  .append('g')                            // grupo de trabajo
  .attr('transform', `translate(${margen.left}, ${margen.top})`);

// ── ESCALAS ───────────────────────────────────────────────────
const x = d3.scaleBand()
  .domain(datos.map(d => d.mes))
  .range([0, ancho])
  .padding(0.2);

const y = d3.scaleLinear()
  .domain([0, d3.max(datos, d => d.valor)])
  .range([altura, 0]);

// ── EJES ──────────────────────────────────────────────────────
svg.append('g')
  .attr('transform', `translate(0, ${altura})`)
  .call(d3.axisBottom(x));

svg.append('g')
  .call(d3.axisLeft(y).ticks(5));

// ── BARRAS ────────────────────────────────────────────────────
svg.selectAll('rect')
  .data(datos)
  .join('rect')
    .attr('x',      d => x(d.mes))
    .attr('y',      d => y(d.valor))
    .attr('width',  x.bandwidth())
    .attr('height', d => altura - y(d.valor))
    .attr('fill',   '#f68e2e')
    .attr('rx',     3); // esquinas redondeadas

líneas

5. Gráfico de líneas

Para dibujar una línea en D3 se usa un generador de línea: d3.line(). Le dices cómo calcular X e Y para cada dato, y te devuelve un string de ruta SVG (d) que se asigna a un <path>. 

// ── GENERADOR DE LÍNEA ────────────────────────────────────────
const linea = d3.line()
  .x(d => x(d.fecha))    // función que devuelve la posición X del punto
  .y(d => y(d.valor))    // función que devuelve la posición Y del punto
  .curve(d3.curveMonotoneX); // curva suavizada (opcional)
  // otras curvas: d3.curveLinear, d3.curveBasis, d3.curveStep

// ── DIBUJAR EL PATH ───────────────────────────────────────────
svg.append('path')
  .datum(datos)             // .datum() para UN elemento que usa todos los datos
  .attr('d', linea)          // el generador produce el string "M x,y L x,y..."
  .attr('fill', 'none')
  .attr('stroke', '#f68e2e')
  .attr('stroke-width', 2.5);

// ── ÁREA RELLENA (bonus) ──────────────────────────────────────
const area = d3.area()
  .x(d => x(d.fecha))
  .y0(altura)               // base del área (fondo del gráfico)
  .y1(d => y(d.valor))      // tope del área (la línea en sí)
  .curve(d3.curveMonotoneX);

svg.append('path')
  .datum(datos)
  .attr('d', area)
  .attr('fill', 'rgba(246,142,46,0.15)')
  .attr('stroke', 'none');

pie / donut

6. Gráfico circular (Pie y Donut)

Para gráficos circulares D3 usa dos generadores: d3.pie() que calcula los ángulos a partir de los datos, y d3.arc() que genera el string SVG de cada sector. 

// ── GENERADORES pie + arc ─────────────────────────────────────
const generadorPie = d3.pie()
  .value(d => d.valor)     // qué propiedad es el valor
  .sort(null);             // null → respetar el orden original

const generadorArc = d3.arc()
  .innerRadius(0)         // 0 = pie lleno; > 0 = donut
  .outerRadius(radio);

// Donut: innerRadius > 0
const arcDonut = d3.arc()
  .innerRadius(radio * 0.55) // agujero del 55%
  .outerRadius(radio);

// ── DIBUJAR ───────────────────────────────────────────────────
// Los datos del pie son los ángulos calculados por d3.pie()
const datosPie = generadorPie(datos); // [{startAngle, endAngle, data, ...}]

svg.selectAll('path')
  .data(datosPie)
  .join('path')
    .attr('d', generadorArc)        // genera el sector SVG
    .attr('fill', d => color(d.data.categoria))
    .attr('stroke', '#fff')
    .attr('stroke-width', 2);

transiciones

7. Transiciones

D3 incluye un sistema de transiciones que anima suavemente los cambios de atributos. Solo hay que insertar .transition() en la cadena antes de los atributos que quieres animar. 

// ── TRANSICIÓN BÁSICA ─────────────────────────────────────────
seleccion
  .transition()              // activa el modo transición
  .duration(800)             // duración en ms
  .delay((d, i) => i * 100) // retardo escalonado por índice
  .ease(d3.easeBounceOut)    // función de easing
  .attr('height', d => y(d))  // este atributo se animará
  .attr('fill', '#4aa8ff');   // y también este

// ── FUNCIONES DE EASING MÁS USADAS ───────────────────────────
d3.easeLinear      // velocidad constante
d3.easeQuadInOut   // suave en inicio y fin
d3.easeCubicInOut  // más suave que quad
d3.easeElasticOut  // efecto resorte al final
d3.easeBounceOut   // efecto rebote al final
d3.easeBackOut     // sobrepasa un poco y vuelve

// ── ACTUALIZAR DATOS CON TRANSICIÓN ──────────────────────────
// El flujo es: nuevos datos → re-bind → transition → update atributos
function actualizar(nuevosDatos) {
  svg.selectAll('rect')
    .data(nuevosDatos)
    .join('rect')
    .transition().duration(600).ease(d3.easeBackOut)
      .attr('y',      d => y(d.valor))
      .attr('height', d => altura - y(d.valor));
}
