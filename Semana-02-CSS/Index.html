:root { } Variables CSS (Custom Properties)

Las variables CSS permiten definir valores una vez y reutilizarlos en todo el documento. Se declaran en :root con el prefijo -- y se usan con var().

/* Declaración en :root */
:root {
  --color-primario: #1a3a4a;
  --espacio-md: 1rem;
  --fuente-titulos: 'Georgia', serif;
}

/* Uso en cualquier selector */
h1 {
  color: var(--color-primario);
  margin-bottom: var(--espacio-md);
  font-family: var(--fuente-titulos);
}

/* Con valor fallback (por si la variable no existe) */
p { color: var(--color-texto, #333333); }
Ventaja principal: Si necesitas cambiar el color primario de toda la página, solo modificas una línea en :root, no buscas en todo el CSS.

Box Model El modelo de caja

Cada elemento HTML es una caja con 4 capas: content → padding → border → margin

.caja {
  /* CONTENT */
  width: 220px;
  height: 80px;

  /* PADDING — espacio interior */
  padding: 16px;                  /* 4 lados iguales */
  padding: 10px 20px;             /* arriba/abajo | izq/der */
  padding-top: 10px;              /* lado individual */

  /* BORDER */
  border: 2px solid #2d7d5f;
  border-radius: 8px;             /* esquinas redondeadas */

  /* MARGIN — espacio exterior */
  margin: 16px;
  margin: 0 auto;                 /* centrar horizontalmente */

  /* Con border-box, width incluye padding + border */
  box-sizing: border-box;
}
box-sizing: border-box
padding y border incluidos en el width
Con outline
no afecta al espacio del layout
Recuerda: margin: 0 auto solo centra elementos block que tienen un width definido. No funciona con inline.

color Formas de definir colores 
/* 1. Nombre de color (solo 147 colores nombrados) */

/* 2. Hexadecimal: #RRGGBB  o  #RGB (abreviado) */
color: #1a3a4a;    /* completo */

color: #1a3a4aaa;  /* con alpha (transparencia) */

color: rgb(26, 58, 74);

color: rgba(26, 58, 74, 0.5);  /* 0=transparente, 1=sólido */

/* 5. HSL — Matiz, Saturación, Luminosidad */
color: hsl(200, 48%, 20%);

/* 6. HSLA — con transparencia */
color: hsla(200, 48%, 20%, 0.5);

position Posicionamiento de elementos
position: sticky → el elemento fluye normalmente hasta que el scroll lo llevaría fuera de la pantalla; entonces se "pega" al viewport. Ejemplo: position: sticky; top: 0; para barras de navegación.
static (defecto)
relative
top:10px left:20px
absolute → top:16px right:16px 
/* Valores de position */
position: static;    /* defecto — flujo normal */
position: relative;  /* se mueve desde su posición original */
position: absolute;  /* relativo al ancestro con position != static */
position: fixed;     /* relativo al viewport, no se mueve al hacer scroll */
position: sticky;    /* híbrido: flujo normal + se pega al scroll */

/* Propiedades de desplazamiento (requieren position != static) */
top: 20px;
right: 0;
bottom: 10px;
left: 50%;

/* z-index: capa de apilamiento */
z-index: 10;    /* mayor número = más al frente */

overflow Control del desbordamiento

visible (defecto)

Este texto es más largo que la caja y se DESBORDA hacia afuera sin recortarse.

hidden

Este texto es más largo que la caja pero se RECORTA y no se ve lo que sobra.

scroll

Este texto es más largo que la caja y aparece un SCROLLBAR para desplazarse, aunque el contenido no desborde.

auto

Este texto es más largo y el scrollbar aparece SOLO si el contenido desborda. Es el más recomendado en general.
Cascada Especificidad en acción

El selector más específico gana la competencia de estilos:

Este párrafo usa la regla general de p → color oscuro.

Con clase .texto-verde → color verde.

Con ID #forzado-rojo → color rojo (ID gana a clase).

/* Especificidad: 1 */
p { color: #1e1e1e; }

/* Especificidad: 10 — gana sobre "p" */
.texto-verde { color: #1a6a3a; }

/* Especificidad: 11 (elemento + clase) */
p.texto-verde { font-style: italic; }

/* Especificidad: 100 — gana sobre todo */
#forzado-rojo { color: #aa2020; }

/* !important — solo en emergencias */
.error { color: red !important; }
